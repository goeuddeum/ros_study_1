## 서비스 통신
클라이언트 --- 서버 둘사이의 통신
특수한 메시지를 즉각적으로 주고 받을때 사용

토픽통신과의 차이점: 한번 통신을 주고 받은 후에 통신을 종료한다.
(topic 은 프로그래밍으로 종료 전까지 계속 유지)
토픽통신
퍼블리셔 -> 토픽 -> 서브스크라이버
서비스통신
서비스서버 ->  서비스응답 -> 서비스클라이언트
서비스서버 <- 서비스요청 <- 서비스 클라이언트 
ros::spin() : 콜백함수전까지 대기
ros::spinOnce(): 호출시점까지 콜백함수를 처리하고 다른거 가능
